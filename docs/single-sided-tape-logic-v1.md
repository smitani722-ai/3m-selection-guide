# 片面テープロジック全面見直しレポート

作成日: 2026-06-18

## 前提

`片面テープロジック.xlsx` を片面テープ選定の Single Source of Truth として扱い、片面テープカテゴリのみ Excel 由来のルート判定へ切り替えた。

基本フロー:

1. Q1 カテゴリー
2. Q2 用途
3. Q3 サブ用途
4. Q4 作業内容
5. Q5 必要性能1
6. Q6 必要性能2
7. 推奨製品

## 現行ロジックとの差異

| 項目 | 旧ロジック | Excelロジック |
|---|---|---|
| 判定方式 | `selectionEngine.ts` 内の個別スコア加点 | Excel由来のルート表を完全優先 |
| 対象範囲 | 主にラインテープ 471 / 5702 / 764 | マスキング、固定、スプライス、保護膜剥離、熱保護、滑り、仮固定、ライン、防水 |
| 質問順 | 全カテゴリ共通の素材・環境・特性質問 | 営業ヒアリング順の用途起点フロー |
| 製品数 | 3製品中心 | 53製品 |
| 到達確認 | 既存営業テスト内の一部のみ | Excel全71ルートを回帰テスト化 |

## 修正内容

| ファイル | 内容 |
|---|---|
| `data/singleSidedTapeRoutes.json` | Excelから抽出した片面テープ71ルートを登録 |
| `lib/singleSidedTapeLogic.ts` | Excelルートの選択肢生成、ルート照合、結果生成を追加 |
| `lib/questions.ts` | 片面テープ選択時だけQ2〜Q6をExcel順に切替 |
| `lib/store.ts` | 片面テープ専用回答フィールドを選定条件へ渡す処理を追加 |
| `lib/selectionEngine.ts` | 片面テープカテゴリのみExcelルート判定を先に実行 |
| `data/products.json` | Excelに登場する片面テープ品番を製品マスターへ追加 |
| `tests/sales-rules.test.ts` | Excel全71ルートの到達テストを追加 |
